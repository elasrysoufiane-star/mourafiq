# .claude/memory — mémoire projet partagée

Notes durables destinées aux sessions Claude Code (et aux humains), versionnées
avec le code. Différence avec CLAUDE.md : CLAUDE.md décrit **l'état actuel**
(architecture, config, commandes) ; ici on garde **le pourquoi** (décisions,
pistes abandonnées, améliorations identifiées mais pas encore faites).

- `decisions.md` — journal des décisions structurantes (une entrée datée par
  décision : quoi, pourquoi, comment revenir dessus).

Règles :
- Écrire une entrée quand une décision non évidente est prise (choix de modèle,
  compromis matériel, piste rejetée) — pas pour ce que le code montre déjà.
- Mettre à jour/supprimer une entrée devenue fausse plutôt que d'en empiler.
